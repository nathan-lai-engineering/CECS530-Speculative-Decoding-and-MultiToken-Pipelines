# Hardware-Assisted Rollback

## Overview
Hardware-assisted rollback accelerates speculative decoding by handling mismatches efficiently at the architectural level.

## Speculative Buffer
A hardware buffer stores speculative tokens generated by the draft model before they are verified.

## Commit Pointer
A commit pointer tracks the last verified token that is safe to keep.

## Rollback Pointer
A rollback pointer marks where incorrect speculative tokens begin.

## Verification Process
- Tokens are compared between draft and target models
- Matching tokens are committed
- On mismatch, rollback is triggered

## Rollback Mechanism
Instead of recomputing from scratch, hardware discards all speculative tokens after the mismatch using the rollback pointer.

## Benefits
- Reduces recomputation cost
- Minimizes pipeline stalls
- Enables faster recovery after incorrect speculation

## Architectural Insight
This approach treats rollback as a control operation rather than a software-level recomputation, improving overall inference efficiency.
